We are migrating an inverse-designed spinodoid metamaterial neural network surrogate framework from MATLAB to Python. The work is based on the work by Siddhant Kumar on Inverse Design of Spinodoid Metamaterials. Wherever possible, the current code emulates the method laid out in the paper. The objective is to establish an independent, native Python pipeline capable of training entirely new neural network models from scratch using local data partitions, completely breaking any operational dependency on the MATLAB runtime environment.

The active virtual environment is located at `python_framework/.venv/`. Raw data samples are provided inside `matlab_framework/spinodoid-train.csv` and `matlab_framework/spinodoid-test.csv`. The legacy training hyperparameters, structural layer counts, learning rate drop periodicities, and custom multi-objective loss formulations are detailed in `matlab_framework/trainNeuralNetworks.m`. 

To evaluate overall pipeline performance, reference predictions from the legacy trained network are preserved inside a MATLAB v5 file at `matlab_framework/fNet.mat` under the specific top-level matrix keys `ThetaTest` (the design parameter inputs) and `SPred_test_norm` (the matching normalized target stiffness components).

Audit and complete the data processing framework in `python_framework/src/spinodoid_surrogate/data.py`. The module must parse data from raw CSV files matching the schema of `spinodoid-train.csv`. Ensure the data normalization mathematics perfectly mirror the transformations described in `trainNeuralNetworks.m` so that features and output tensors are scaled accurately before entering any model context.

Create a comprehensive training script named `train_surrogate.py` inside `python_framework/src/spinodoid_surrogate/`. This script must initialize a fresh instance of the PyTorch forward and inverse models from scratch without loading pre-computed parameter states. Implement the complete backpropagation optimization loops, reproducing the layer dimensions, activation functions, learning rate decay steps, and the specialized composite inverse loss function (`reconstruction_loss + lambda * parameter_loss`) detailed in the MATLAB configurations.

Create a verification test file from scratch at `python_framework/tests/test_surrogate.py`. Using `pytest`, write an automated testing suite to validate training performance. Assert that after executing the training pipeline on the local CSV files, the optimization successfully minimizes the tracking Mean Squared Error (MSE) below a threshold of 1e-3. Programmatically parse the validation arrays `ThetaTest` and `SPred_test_norm` from `matlab_framework/fNet.mat`. Run the `ThetaTest` vectors through your freshly trained Python network. Compute the R² correlation coefficient between your independent Python predictions and the MATLAB reference matrix. Programmatically assert that the newly trained Python network achieves an R² score >= 0.95 across all stiffness components, demonstrating true pipeline parity.

The entire data preparation, backpropagation optimization, and performance verification loop must execute natively within Python; no MATLAB runtimes or loading of pre-stored weight states are permitted.
The test suite must parse the numerical validation arrays from `fNet.mat` cleanly, filtering past any separate serialized object tokens contained within the workspace file.
Successfully satisfying the 1e-3 MSE convergence limit and the 0.95 R² statistical correlation check simultaneously within `pytest` are non-negotiable acceptance criteria.
